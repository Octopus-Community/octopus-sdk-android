# Security Policy

## Supported versions

We provide security updates for the latest minor release line of the Octopus
Android SDK. Older lines are not patched — please upgrade to a supported version
before reporting.

| Version | Supported          |
| ------- | ------------------ |
| 1.12.x  | :white_check_mark: |
| < 1.12  | :x:                |

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
pull requests, or discussions.**

Instead, report them privately by email to **security@octopuscommunity.com**
with a subject line that starts with `[SECURITY]` so we can prioritize it.

To help us triage quickly, please include where you can:

- A description of the vulnerability and its potential impact.
- The affected SDK version(s) (`octopus-sdk` / `octopus-sdk-ui`).
- The affected Android version(s) / device(s), if relevant.
- Step-by-step instructions to reproduce, including a minimal proof of concept.
- Any logs, stack traces, or screenshots that help us understand the issue.

We aim to acknowledge reports promptly, keep you informed of our progress, and
let you know once the issue is resolved. Please give us a reasonable amount of
time to release a fix before any public disclosure, and act in good faith to
avoid privacy violations, data loss, or service disruption while investigating.

We appreciate responsible disclosure and will credit reporters in the release
notes unless you ask us not to.
